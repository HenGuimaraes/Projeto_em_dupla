Para enviar os arquivos para o github e commitar tem 3 passos:
a) adicionar os arquivos que você quer antes de serem commitados:
   git add - para adicionar um arquivo especifico ou vários especificos então: git add nomeDoArquivo.extensão; 
   caso seja todos então: git add .

b) commitar, é dizer o porque você esta adicionando aqueles arquivos antes de manda-los para o github(internet):
   git commit -m "descrição do porque"
o -m é para dizer que você vai por uma descrição

c) push, para mandar os arquivos para o github(internet), vai pedir sua senha ou para você se logar, ai você loga:
   git push

O contribuidor sempre tem que puxar o que tem no github que ele ainda não tem e depois fazer o push, para puxar:
   git pull
e ai o git push vai funcionar;

BRANCH E PR

Branch são tarefas que você pode criar para não se misturar com a versão "oficial" do projeto, assim você cria uma versão virtual para poder ser modificado
PR é o pull request que você faz para poder subir sua tarefa, antes de subir com o pull request você vê o que foi alterado e se o seu código vai para produção

Para criar um branch você tem que ir no seu projeto no site do github e criar lá na caixinha de branch, depois que criado, vá no seu gitbash e de um git pull para seu projeto saber das novidades
Para mudar de branch: git checkout nomeDaTarefa

Tudo que foi feito naquele branch ficara apenas naquele branch, para misturar você tem que ir no site do github e dar um pull request, você pode ver as alterações e depois confirmar o pull request e ai você pode deixar lá para que os outros revisem antes de dar o merge e misturar os arquivos a outro branch, ou seja, o branch oficial.
